## 3. Manual script

Intended script:

1. Title → New Run.
2. Run HUD shows real store state.
3. Drag a movable item to an empty slot; `moves.freeRemaining` decrements.
4. Exhaust free moves; paid moves spend coins, unaffordable moves reject and shelf
   returns to store state.
5. Open Shop; day advances and `lastScoringTrace.traceId` includes `-d${day}`.
6. Return to Title → Continue; active save restores the same state hash.

Manual web was **not completed in this environment**. Expo CLI printed
`Networking has been disabled` and `Starting project...`, but never bound port 8090.
`lsof -i :8090` was empty and `curl -I http://localhost:8090` failed. Tried:

- `node_modules/.bin/expo start --web --port 8090 --host localhost`
- `node_modules/.bin/expo start --web --offline --port 8090`
- `/usr/bin/env EXPO_NO_TELEMETRY=1 CI=1 node_modules/.bin/expo start --web --offline --port 8090`

## 4. Known issues + spec deviations

- Web manual loop needs rerun in a working Expo/Metro environment; port 8090 would not
  bind here.
- `startNewRun()` currently seeds an M2 arrange-phase starter state so the existing HUD
  can immediately exercise drag/move economy before Lane B's draft/restock screens are
  integrated. `startFreshRun()` is present for true `createRun(seed)` initialization.
- Run HUD primary actions for delivery/restock use first-offer/first-empty-slot
  scaffolding only. Lane B's M2 draft/restock screens should replace that presentation.
- Active-run persistence stores the authoritative `GameState` snapshot for Continue.
  Replay action-log persistence is still sim-level, not included in the active save.
